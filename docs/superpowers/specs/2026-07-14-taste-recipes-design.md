# イメージテイスト拡充 & テイスト別配色レシピ — 設計書

日付: 2026-07-14 / 承認済み

## 目的

1. イメージテイストを 30 → 40 種類に拡充する
2. 配色結果を色相理論ベースの固定 6 スキーム（同系色・類似色・補色・スプリット・トライアド・トーングラデーション）から、**選択テイストの世界観に合った固有の配色レシピ 6 パターン**に置き換える
3. GitHub（1935Hiro/color-scheme, public）で管理する

## 方式（承認済みの選択）

- 生成方式: **基準色 × テイスト別レシピ**（基準色を活かしつつテイスト固有のレシピで生成）
- テイスト数: 40 種類前後
- GitHub: gh CLI をインストールし public リポジトリを作成

## 配色エンジン

### レシピ DSL

各テイストは `recipes`（6 個）を持つ。各レシピは `name`（世界観に合った名前）と `colors`（6 スペック）を持つ。

スペックのフィールド:

| キー | 意味 |
|---|---|
| `b: 1` | 基準色そのまま（各レシピに必ず 1 つ含める） |
| `dh` | 基準色からの相対色相（度） |
| `hue` | テイスト固有の絶対色相アンカー。`blend`（既定 0.25）の割合で基準色相へ引き寄せる |
| `n: 1` | ニュートラル色。彩度を低く固定し、明度は絶対レンジ（約 12–94%）へマップ |
| `s` / `l` | テイストのトーンレンジ内の相対位置（0–1）。範囲外も許容し最終値でクランプ |

### 基準色との連動

- 色相: `hue` アンカーは基準色相へ 25% 程度ブレンド。`dh` は基準色相からの相対オフセット
- 彩度・明度: 基準色の S/L の 50% からの偏差 × 0.25 をレシピ位置に加算（スライダー操作が全パレットに緩やかに反映される）

### 廃止するもの

- `SCHEMES` 配列（色相理論ベースの固定スキーム）
- `TASTES` の `huePull`（レシピの `hue` アンカーが役割を代替）

## ファイル構成

- `tastes.js`（新規）— 40 テイスト定義＋240 レシピのデータ
- `app.js` — `generatePalettes` をレシピベースに書き換え。UI・アクセシビリティ・メタリック集は既存のまま
- `index.html` — `tastes.js` を `app.js` より先に読み込む
- `scripts/check.js`（新規）— データ検証（テイスト数・レシピ数・基準色の包含・hex 妥当性）
- `README.md`（新規）

CVD シミュレーションは `generatePalettes` を共有しているため自動で新配色に追従する。ドリルダウン・基準色バッジ・コピー機能は変更なし。
